**Overall Layout & Aesthetic**
The website is designed as a full-screen, single-page application mimicking a modern, dark-mode IDE (Integrated Development Environment) like VS Code. The color palette utilizes deep charcoal backgrounds, neon syntax-highlighting colors (pinks, cyans, and greens), and sleek glassmorphism effects for floating elements. 

At the top, a traditional IDE tab reads `portfolio.ts — Unsaved Changes`. Below this, the Header presents the developer's name as a declared variable: `const developer = "Geva Kipper"; // Software Engineer at Google currently building [Google team/project description].` 
Immediately beneath this, styled as a prominent IDE comment or warning banner, is the subtitle: `/* WARNING: This website is re-designed each day by AI and is thus ever-changing. */`

**The Gamification / Debugging Mechanic**
The core experience is that the portfolio is initially "broken." The central workspace contains an 8-box "Bento Grid" layout representing the sections, but initially, these boxes are entirely rendered as raw, visually glitched code blocks with red squiggly lines underneath "syntax errors." 

To view the actual portfolio, visitors must "debug" the site. Hovering over a red squiggly line summons a sleek tooltip reading `Quick Fix: Compile UI`. Clicking this triggers a smooth, modern CSS animation where the raw code collapses, a terminal sound plays, and a beautiful, fully visual graphic card fades into its place. The user must debug all 8 boxes to complete the layout.

**The 8 Bento Grid Modules (Once Debugged)**

**Box 1: GlglStats**
Design: A music-themed card featuring a neon audio waveform graphic. 
Content: A brief description highlighting the Python and LZ77 compression algorithm used to measure and visualize repetitiveness in Israeli pop-chart songs.
Interactions: Two sleek, glowing buttons. One links directly to the app (https://glglstats.geekip.dev) and an embedded styled thumbnail acts as a link to the PyData TLV presentation (https://www.youtube.com/watch?v=jrKAtOsj1Lo).

**Box 2: Ahuzat Dibuk**
Design: A visual backdrop resembling a Tel-Aviv map overlaid with a glowing heat-map. 
Content: Text explaining the Python scraping and data visualization of parking lot occupancies. 
Interactions: Three stylized "News Output Logs" that act as outbound links to Channel 13 (https://13tv.co.il/item/news/domestic/internal/parking-902956407/), Mako (https://www.mako.co.il/nexter-news/Article-ae43964b891bf71027.htm), and the original Facebook post (https://www.facebook.com/groups/secrettelaviv/posts/10159636301195943/).

**Box 3: 2D Part Orienting**
Design: A highly structural, geometric aesthetic showing 2D wireframe polygons to represent computational geometry.
Content: A summary of the C++ implementation for constructing "push plans" to orient polygons.
Interactions: A clean, technical-looking button routing to the TAU CGL project documentation (https://www.cgl.cs.tau.ac.il/projects/2d-part-orienting/).

**Box 4: The Turing Jest**
Design: A split-screen, vs-arena aesthetic (e.g., Red vs. Blue sides) mimicking a competitive leaderboard.
Content: Description of the LMSYS-like arena where LLMs compete at making and explaining puns, rated by users.
Interactions: A prominent "Enter the Arena" button linking to https://jest.geekip.dev.

**Box 5: Rhymes of Israel**
Design: A card that visually transitions from a serious, monochromatic newspaper headline to a colorful, comedic highschool yearbook design.
Content: Explains the Chrome extension using an LLM to rewrite Israeli news.
Interactions: A distinct "Add to Chrome" CTA linking out to the Chrome Web Store (https://chromewebstore.google.com/detail/eainginjgdkpgolpjniijdihfbeeeklb).

**Box 6: Hatul Basak**
Design: A terminal-pixel-art aesthetic mixed with a Wikipedia puzzle icon motif. 
Content: Text presenting the Hebrew port of catfishing.net, emphasizing the AI-generated daily challenges and user-created themed Wikipedia trivia puzzles.
Interactions: A "Play & Create" button taking users to https://hatulbasak.geekip.dev.

**Box 7: Meta Portfolio Project**
Design: A recursive, "infinite mirror" visual effect. The card contains a miniature, slightly skewed live-preview of the very website the user is currently debugging.
Content: Text playfully pointing out the meta nature of the project.
Interactions: A self-referential link pointing to https://geekip.dev.

**Box 8: System Executables (Socials)**
Design: A minimalist control panel layout. 
Content: Three sleek, interactive nodes or icons representing social platforms.
Interactions: Links acting as executable scripts: `run(github.com/gevak)`, `run(linkedin.com/in/gevakip)`, and `run(facebook.com/geva.kipper)`.

**Bottom Pane: The Terminal / Footer**
Fixed to the bottom of the screen is an IDE terminal panel titled `Debug Console`. It features standard green monospace terminal text on a black background. 
The console continually prints out blinking cursor text: 
`> Compilation complete. Site generated by AI Model [Insert Your Chosen AI Name Here].`
Directly beneath this, a live, ticking stopwatch processes the uptime:
`> System Uptime since 2026-05-26 02:36:22: [HH:MM:SS live counter dynamically updating every second].`
