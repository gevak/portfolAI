Here is the design specification for the portfolio website. 

### Overall Theme & Layout
The website will be designed as a sleek, interactive, full-screen IDE (Integrated Development Environment) clone, heavily inspired by modern themes like VS Code's "Dracula" or "One Dark Pro". The primary background is a deep charcoal blue/grey setting, utilizing a high-contrast syntax highlighting color palette (neon pinks, cyans, greens, and yellows) for typography. All textual elements should use a clean, monospaced font like Fira Code or JetBrains Mono. 

The screen is split into three main areas: a Left Sidebar (File Explorer), a Central Main Editor containing the Header and Project Boxes, and a Bottom Status Bar.

### 1. Header Array: The "Docstring"
At the top of the main editor, replacing a standard website header, design a large Python-style comment block (`"""`). 
*   **Developer Info:** Displayed as formatted variables: `const developer = "Geva Kipper";` and `const currentRole = "Software Engineer @ Google";`.
*   **Subtitle:** Displayed as a brightly colored compiler warning: `WARNING: This UI is completely re-designed each day by an AI agent. Expect ever-changing layouts and continuous mutations.`

### 2. The Main Content: The Project Grid (The 8 Boxes)
The main coding area will feature a masonry or grid layout of 8 distinct "Windows" or "Editor Tabs". Each box should look like a floating code-snippet pane with a tab at the top (e.g., `glglstats.py`, `orienting.cpp`). Hovering over a box should add a soft neon glow around its borders.

*   **Box 1: glglstats.py (GlglStats)**
    *   **Visual:** Python icon on the tab. Content features a mock data visualization chart rendered in ASCII or stylized CSS.
    *   **Content:** Text explaining the LZ77 compression algorithm used to measure and visualize repetitiveness in Israeli pop-chart songs.
    *   **Actions:** Two prominent IDE "Run" buttons. One links to the live site (`https://glglstats.geekip.dev`), the other links to the PyData TLV presentation (`https://www.youtube.com/watch?v=jrKAtOsj1Lo`).

*   **Box 2: ahuzat_dibuk.py (Ahuzat Dibuk)**
    *   **Visual:** Jupyter Notebook style cell indicating scraping and heat-map generation for Tel-Aviv parking lot occupancies. 
    *   **Content:** Project description styled as terminal output. 
    *   **Actions:** "Press Coverage" links styled as clickable terminal hyperlinks pointing to Channel 13 (`https://13tv.co.il/item/news/domestic/internal/parking-902956407/`), Mako (`https://www.mako.co.il/nexter-news/Article-ae43964b891bf71027.htm`), and the Secret Tel-Aviv Facebook post (`https://www.facebook.com/groups/secrettelaviv/posts/10159636301195943/`).

*   **Box 3: orienting.cpp (2D Part Orienting)**
    *   **Visual:** C++ logo tab. Background features faint geometric wireframes of polygons.
    *   **Content:** Description of the algorithm constructing a 'push plan' to orient polygons, implemented in an open-source computational geometry library.
    *   **Actions:** A clickable Git commit link button pointing to the computational geometry library page (`https://www.cgl.cs.tau.ac.il/projects/2d-part-orienting/`).

*   **Box 4: turing_jest.js (The Turing Jest)**
    *   **Visual:** JavaScript icon tab. 
    *   **Content:** Text explaining the LMSYS-like arena framework where LLMs compete to make and explain puns, rated by users. 
    *   **Actions:** A flashy, glowing `[Execute: Enter Arena]` button linking to `https://jest.geekip.dev`.

*   **Box 5: rhymes_of_israel.json (Rhymes of Israel)**
    *   **Visual:** A JSON config or Chrome Extension manifest file styling.
    *   **Content:** Brief on the LLM tool rewriting Israeli news headlines into highschool yearbook-style comedic rhymes.
    *   **Actions:** A "Download Plugin" button directly linking to the Chrome Web Store (`https://chromewebstore.google.com/detail/eainginjgdkpgolpjniijdihfbeeeklb`).

*   **Box 6: hatul_basak.ts (Hatul Basak)**
    *   **Visual:** TypeScript logo tab. 
    *   **Content:** Details exactly how it acts as a Hebrew port of catfishing.net, generating daily Wikipedia trivia challenges via AI, combined with user-generated themes.
    *   **Actions:** Two interactive buttons styled as functions: `playDaily()` and `createPuzzle()` both linking to `https://hatulbasak.geekip.dev`.

*   **Box 7: portfolio.html (Meta Portfolio)**
    *   **Visual:** HTML logo. Styled as the current DOM tree of the site itself (a recursive joke).
    *   **Content:** "A meta self-referential project displaying the exact site you are currently viewing."
    *   **Actions:** A recursive link pointing back to `https://geekip.dev`.

*   **Box 8: secrets.env (Connections)**
    *   **Visual:** Dotenv file format. 
    *   **Content:** Socials mapped out as key-value pairs:
        `GITHUB="github.com/gevak"`
        `LINKEDIN="linkedin.com/in/gevakip"`
        `FACEBOOK="facebook.com/geva.kipper"`
    *   **Actions:** Each string value should be a clickable, richly-colored hyperlink opening the respective social profile.

### 3. Footer: The IDE Status Bar
At the very bottom of the screen, design a thin, distinctly colored (e.g., solid vivid blue or purple) horizontal status bar, identical to an IDE's bottom panel.
*   **Left Side aligned:** "Compiled by: [Insert LLM Name] - AI Architect Agent." (Credits to the AI).
*   **Right Side aligned:** A blinking, live digital stopwatch component running via JavaScript. It must compute the delta between the current real-time and the anchor date `2026-07-20 03:25:39`, displaying the output in exact `HH:MM:SS` (or `DD:HH:MM:SS`) since the chronological anchor.

### 4. Interactive Touches
Include realistic code-editor flair: a blinking text cursor at the end of the header text, line numbers running down the left side of every individual project box, and scroll-bars styled like minimalist IDE scrubbers. Text selection across the site should highlight in a bright, contrasting neon hue.
