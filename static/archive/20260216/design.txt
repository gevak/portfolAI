# Portfolio Website Design: "GevaOS"

## Overall Concept

The entire page presents itself as a retro-modern operating system desktop environment ‚Äî think a blend between Windows 95's charm and a sleek modern Linux tiling window manager. The background is a deep gradient wallpaper shifting from midnight navy to a muted teal, overlaid with a subtle animated grid of faintly glowing dots that slowly pulse, evoking a circuit board or starfield. The aesthetic is flat-design with pixel-art accents: sharp window borders, monospaced typography mixed with clean sans-serif fonts, and a muted but warm color palette (soft amber, dusty rose, terminal green, slate blue, cream white).

## Top Bar (Taskbar)

A fixed horizontal taskbar sits at the very top of the viewport, styled like a classic OS menu bar. On the left it reads "GevaOS v2025.02" in a small monospaced font. On the right, a tiny real-time clock displays the current time, and beside it a small animated Wi-Fi icon and a battery icon stuck at 100%. The center of the taskbar displays in a gentle typewriter animation: **"‚ö† This desktop is redesigned each day by AI ‚Äî no two visits are ever quite the same."** The text has a subtle shimmer effect, as if the pixels are gently rearranging themselves.

## Desktop Greeting Area

Below the taskbar, centered on the upper portion of the desktop, is a "Welcome" dialog box ‚Äî a draggable-looking window (though it stays fixed) with a title bar reading "Welcome.exe". Inside, a clean profile section: the name **"Geva Kipper"** rendered large in a bold, slightly rounded sans-serif font (like Inter or DM Sans) with a subtle text-shadow glow in amber. Below it, in a lighter weight: **"Software Engineer at Google, building things that make the web faster."** The window has a small animated pixel-art penguin or robot mascot in the corner, blinking occasionally.

## Desktop Icons & Folder Windows

Scattered naturally across the desktop below the welcome dialog are seven clickable desktop icons arranged in a loose, organic grid ‚Äî not perfectly aligned, as if someone casually placed them on their desktop. Each icon is a stylized folder or application icon rendered in a chunky pixel-art style with a label underneath in small white monospaced text. When you hover over any icon, it gently bounces and its label highlights in amber.

**Clicking an icon opens a "window"** that slides up and expands from the icon's position with a smooth spring animation. Each window has a classic title bar with a colored close button (red circle), a minimize button (yellow circle), and a maximize button (green circle). The close button actually closes/collapses the window back to its icon. Multiple windows can be open simultaneously and they layer with z-index stacking ‚Äî clicking a window brings it to front. Each window has a very subtle drop shadow and a thin 1px border in a lighter shade of its accent color.

## Interactions & Polish

All windows start in a closed/minimized state so the user first sees the desktop with scattered icons and the welcome dialog. Clicking an icon opens its window with a satisfying spring animation (CSS transitions with slight overshoot). The desktop icons can optionally have a very subtle parallax drift when the mouse moves across the page, giving a sense of depth. The entire page uses smooth scrolling if the content exceeds viewport height on smaller screens. On mobile, the layout adapts: the desktop icons stack into a vertical scrollable list, and tapping one expands its content inline accordion-style rather than as floating windows, since draggable windows don't suit touch interfaces. The color scheme respects `prefers-color-scheme`: the dark theme described above is default, but a light mode inverts to a creamy off-white desktop with pastel window accents and darker text if the user's system is set to light mode. A tiny theme toggle icon (sun/moon) in the taskbar allows manual switching.
