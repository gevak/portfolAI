# Terminal Portfolio Design: "GEVA_OS v2.0"

## Overall Concept & Visual Identity

The website presents itself as a retro-futuristic operating system boot sequence that has evolved beyond simple text. The background is a deep space black (#0a0a0f) with a subtle animated grid of faint cyan lines that slowly pulse, suggesting an active system. The entire page has a slight CRT screen curvature effect applied via CSS, with a very subtle scanline overlay and occasional screen flicker animation that triggers randomly every 30-60 seconds.

The primary color palette draws from classic terminal aesthetics but elevated: phosphor green (#33ff66) for primary text and commands, electric cyan (#00d4ff) for links and interactive elements, amber (#ffaa00) for warnings and highlights, and soft magenta (#ff66aa) for special accents. All text uses a monospace font stack: "JetBrains Mono" with "Fira Code" as fallback.

Upon page load, a boot sequence animation plays: lines of system diagnostics rapidly scroll for 2 seconds ("INITIALIZING NEURAL CORES... LOADING PERSONALITY MATRIX... The developer's name "GEVA KIPPER" appears as if being typed character by character in large ASCII art letters made of the = and | characters, approximately 80px equivalent height, centered at the top.

Below the ASCII name, in standard terminal text: ">> CURRENT_STATUS: Engineering @ Google | Building things that sometimes work" ‚Äî this line has a blinking cursor at the end that persists throughout the user's visit.

Beneath this, a special styled block appears with a dashed border made of ~ characters: "‚ö† NOTICE: This interface is procedurally regenerated daily by AI consciousness. What you see today will never exist again. Reality version: [current date]" ‚Äî the date portion pulses gently.

## Command Line Interface

Below the header sits a functional command input field styled as a classic terminal prompt: "visitor@geva-os:~$ " followed by an input field with a blinking block cursor. When users type and press enter, commands trigger smooth scrolls to respective sections. Valid commands displayed in a subtle "hint" row below: "help | projects | glglstats | ahuzat | orienting | jest | rhymes | meta | connect | clear". Invalid commands return humorous error messages that fade after 3 seconds ("Error 418: I'm a teapot, not a mind reader. Try 'help'").

The six project boxes are arranged in a staggered asymmetric grid ‚Äî two columns on desktop, but the boxes have varying heights and slight horizontal offsets (some pushed 20px left, others 20px right) creating visual rhythm. Each box appears to be a separate terminal window floating above the background, complete with a title bar showing a mock window control (three dots: red, yellow, green) and the project name.

The footer sits at the very bottom, separated by a line of repeating ‚ïê characters. Centered text reads: "// Interface rendered by Claude (Anthropic) | Model: Opus" with a small sparkle animation on "Claude". 

Below this, a live stopwatch component displays in larger monospace text: "SYSTEM UPTIME SINCE 2025-12-08 10:57:28" followed by a dynamically updating counter showing "XXXXd XXh XXm XXs" ‚Äî each number segment is contained in its own subtle box, and the seconds actively count up, creating constant motion on the page. The entire counter has a subtle glow effect in cyan.

## Interactions & Micro-animations

All boxes have a "typed reveal" animation when they first enter the viewport ‚Äî content appears character by character at high speed (50ms per character) as if being printed to terminal. Hovering over any box causes a subtle "selection" effect ‚Äî a cyan border glow pulses once, and the box lifts slightly (transform: translateY(-4px)) with a soft shadow.

Links throughout change from their base color to white on hover, with the transition styled as a "glitch" ‚Äî the text briefly offsets horizontally by 2px in alternating directions before settling.

The cursor throughout the page (when hovering over interactive elements) changes to a block cursor shape rather than a pointer, maintaining the terminal aesthetic.

## Mobile Adaptation

On mobile, the staggered grid becomes a single column. The command input transforms into a dropdown menu of sections for easier navigation. The ASCII art name scales down to a simpler text-based header. All terminal window chrome (the title bars with dots) remains but becomes more compact. The boot sequence shortens to 1 second.
